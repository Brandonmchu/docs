## Overview

Proposals let you collaborate with clients before invoicing. Build structured estimates, send professional documents, track status, and convert approved proposals into invoices with one click.

**Why Use Proposals:**
- Get client approval before doing work
- Set clear expectations
- Professional presentation
- Seamless conversion to invoices
- Track approval status
- Reduces billing disputes

## Proposal vs Invoice

**Proposals** are pre-work estimates:
- Sent before work begins
- Client reviews and approves
- Can be negotiated and revised
- Not a demand for payment
- Becomes invoice after approval

**Invoices** are post-work payment requests:
- Sent after work complete (or milestone)
- Demands payment
- Less negotiable
- Legal document

<Tip>
  Use proposals for new clients, large projects, or work where scope needs approval first.
</Tip>

## Creating Proposals

### Basic Proposal Creation

1. Navigate to **Proposals**
2. Click **Create Proposal** or **+** button
3. Fill in proposal details

**Required:**
- **Client**: Who you're proposing to
- **Title**: Proposal name/summary
- **Line Items**: What you're proposing to do

**Optional:**
- **Scope**: Detailed description of work
- **Client Message**: Personal note to client
- **Media**: Images, mockups, examples
- **Attachments**: Contracts, specifications
- **Terms**: Payment terms, conditions
- **Expiration Date**: When proposal expires

4. **Save as Draft** or **Send Proposal**

### Proposal Components

#### Title & Scope

**Title:**
- Short, descriptive name
- Example: "Website Redesign Proposal"
- Appears in subject line and document header

**Scope:**
- Detailed description of work
- What's included
- What's not included
- Timeline and milestones
- Deliverables
- Success criteria

**Good Scope Example:**
```
Complete website redesign including:
- 5 page designs (Home, About, Services, Portfolio, Contact)
- Mobile-responsive layouts
- Custom branding integration
- SEO optimization
- 2 rounds of revisions
- Training on content management

Timeline: 6 weeks from approval
Deliverables: Design files, live website, training video

Not included: Content writing, photography, ongoing maintenance
```

#### Client Message

Personal message to accompany proposal:

**Use For:**
- Friendly greeting
- Context about the proposal
- Why you're excited about project
- Next steps
- Call to action

**Example:**
```
Hi Sarah,

I'm excited to present this proposal for your website redesign.
Based on our call last week, I've outlined everything we discussed
plus a few ideas I think would really make your site shine.

Take a look and let me know if you have any questions. I'm happy
to jump on a call to discuss any aspect of the proposal.

Looking forward to working together!

Best,
[Your Name]
```

#### Line Items

Same powerful line item system as invoices:

**From Offerings:**
- Select pre-defined services/products
- Consistent pricing
- Professional descriptions

**Manual Entry:**
- Custom line items
- Flexible descriptions
- Per-project pricing

**Line Item Features:**
- Description, quantity, rate
- Tax application
- Subtotal calculation
- Discounts
- Total amount

#### Media & Attachments

**Media** (Inline Images):
- Mockups
- Design samples
- Example work
- Portfolio pieces
- Embedded in proposal document

**Attachments** (Downloadable Files):
- Detailed specifications
- Contracts to sign
- Terms and conditions
- Reference materials
- Supporting documents

**Supported Formats:**
- Images: JPG, PNG
- Documents: PDF, DOCX
- Up to 5MB per file

## Proposal Status Flow

### Draft
- Work in progress
- Not sent to client
- Can edit freely
- Not visible to client

### Issued
- Sent to client
- Client can view
- Awaiting response
- Public proposal page available

### Approved
- Client accepted proposal
- Work can begin
- Ready to convert to invoice
- Approval details captured (signer name, email, timestamp)

### Rejected
- Client declined proposal
- No work to be done
- Preserved for records
- Can revise and resend as new proposal

### Void
- Cancelled/obsolete
- No longer valid
- Preserved for history

<Note>
  Unlike invoices, proposals can be revised and reissued if client requests changes.
</Note>

## Sending Proposals

### Delivery

**To Send Proposal:**
1. Complete proposal
2. Review all details
3. Click **Send Proposal**
4. Confirm send

**What Happens:**
- Email sent to client
- Includes proposal link
- Professional branded message
- Client can view and approve online
- Email tracking begins

### Public Proposal Pages

Each proposal gets secure, branded page:

**Features:**
- Organization branding (logo, colors)
- Full proposal details
- Line items and pricing
- Scope and terms
- Media and attachments
- **Approve Proposal** button
- Mobile-responsive
- Print-friendly

**Security:**
- Unique URL
- No login required
- HTTPS secure
- Optional expiration

### Email Tracking

Track proposal engagement:

**Tracked Events:**
- Email delivered
- Email opened
- Link clicked
- Proposal viewed
- Approved/rejected

**Benefits:**
- Know when client viewed proposal
- Time follow-ups appropriately
- Measure engagement
- Identify interested prospects

## Approval Process

### Client Approval

**Client Side:**
1. Receives proposal email
2. Clicks link to view proposal
3. Reviews all details
4. Clicks **Approve Proposal** button
5. Enters name and email (confirmation)
6. Submits approval

**Your Side:**
- Receive approval notification
- Approval details recorded
- Proposal status changes to Approved
- Ready to convert to invoice

**Approval Details Captured:**
- Approver name
- Approver email
- Timestamp
- IP address (audit trail)

<Tip>
  Captured approval creates clear record of client agreement before work begins!
</Tip>

## Converting Proposals to Invoices

One of the most powerful features:

### Conversion Process

**To Convert:**
1. Open approved proposal
2. Click **Convert to Invoice**
3. New invoice created with:
   - Same client
   - Same line items (prices, quantities, descriptions)
   - Same attachments
   - Proposal reference linked
4. Review invoice
5. Adjust if needed (dates, payment terms)
6. Send invoice

**What's Preserved:**
- All line items
- Client information
- Attachments
- Line item descriptions and pricing
- Discounts and taxes

**What's Not Copied:**
- Scope description (not needed on invoice)
- Client message (proposal-specific)
- Media (unless also attached)

<Tip>
  Converting proposal to invoice takes seconds. No retyping line items or amounts!
</Tip>

## Managing Proposals

### Proposal List View

**Features:**
- All proposals displayed
- Status filters (draft, issued, approved, rejected)
- Search by client or title
- Sort by date or amount
- Summary cards (total value, approval rate)

### Editing Proposals

**Draft Proposals:**
- Edit freely
- All fields editable

**Issued Proposals:**
- Limited editing (preserve what client saw)
- Can add notes
- Can't change amounts
- Create new version if major changes needed

### Duplicating Proposals

Reuse proposals for similar projects:

**To Duplicate:**
1. Open proposal
2. Click **Duplicate**
3. New draft created
4. Edit for new client/project
5. Send

**Use Cases:**
- Similar projects for different clients
- Same service package offered to multiple clients
- Revised proposal after feedback

### Voiding Proposals

Cancel obsolete proposals:

**To Void:**
1. Open proposal
2. Click **Void Proposal**
3. Add reason (optional)
4. Confirm

**When to Void:**
- Client went with competitor
- Project cancelled
- Superseded by revised proposal
- No longer relevant

## Common Workflows

### Full Proposal-to-Invoice Flow

1. Prospect inquiry
2. Discovery call
3. Create proposal with scope and pricing
4. Send proposal
5. Track when client opens it
6. Follow up after 3 days
7. Client approves
8. Convert to invoice
9. Begin work
10. Complete work
11. Client pays invoice

### Revised Proposal After Negotiation

1. Send initial proposal
2. Client requests changes
3. Duplicate original proposal
4. Adjust scope and/or pricing
5. Add note: "Revised per our discussion"
6. Send revised proposal
7. Client approves
8. Convert to invoice

## Troubleshooting

### Client Can't Approve Proposal

**Check:**
- Proposal is issued (not draft)
- Link is correct
- Not expired
- Browser allows form submission

### Conversion Doesn't Copy Everything

**Normal:** Scope and client message are proposal-specific and don't copy.

**Solution:** Line items, attachments, and client info DO copy. Add invoice-specific notes separately.
